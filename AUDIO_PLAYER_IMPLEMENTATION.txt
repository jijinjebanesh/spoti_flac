HOW IT WORKS:
─────────────

🔄 PLAYBACK FLOW:
   User taps "Play" → playTrack() in PlaybackController
   ↓
   Audio service initialized if not already
   ↓
   playTrack() creates MediaItem (title, artist, album, cover art)
   ↓
   Audio file loaded via just_audio
   ↓
   Background notification appears (Android lock screen)
   ↓
   In-app AudioPlayerNotification widget appears
   ↓
   User can: play/pause from notification or mini-player
   ↓
   Background playback continues even when app minimized

TROUBLESHOOTING:
────────────────

Issue: Notification doesn't appear
→ Check AudioService initialization in audio_service_provider.dart
→ Verify Android notification permissions
→ Check: AudioServiceConfig.androidNotificationChannelId matches

Issue: Background playback stops
→ Ensure AudioServiceNotifier._init() completes successfully
→ Check Android battery optimization settings
→ Verify foregroundServiceType="mediaPlayback" in manifest

Issue: Cover art not showing
→ Verify coverArtUrl is valid URL
→ Check image loading permissions
→ Try with different image URL

Issue: State not updating
→ Verify audioServiceProvider is being watched/read correctly
→ Check Riverpod provider configuration
→ Ensure playbackEventStream listeners are active
