KEY ARCHITECTURAL INSIGHTS
═════════════════════════════════════════════════════════════════════════════

The Fix Explained Simply:
──────────────────────────
   ❌ BEFORE: Two teams working on same project with no communication
      - Handler team: managing one AudioPlayer
      - Controller team: managing their own AudioPlayer
      - Result: Chaos, conflict, failure

   ✅ AFTER: One team, one project, shared resources
      - Handler team: owns AudioPlayer
      - Controller team: borrows from handler
      - Result: Perfect coordination, works as intended

The Pattern:
──────────────
   1. OWNERSHIP: Handler owns the player (not shared, not copied)
   2. REFERENCE: Controller gets reference via getPlayer()
   3. TIMING: Set mediaItem BEFORE calling play()
   4. SYNC: Sync mediaItem on every state change
   5. CONTROL: All playback goes through the same player
